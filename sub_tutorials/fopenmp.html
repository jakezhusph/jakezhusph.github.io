fopenmp option of clang error

Jiaqiang Zhu, 01/27/2020

When I try to install R package SPARK on my mac laptop, I got the following error under macOS:
clang: error: unsupported option '-fopenmp'
With Google, I found two posts (links listed at the end) and solved the problems.
This is just a note for future use:

  1. Installed xcode from Apple Store
  2. Installed llvm : brew install llvm
  3. Downloaded and installed the gfortran binary installer. Note: You will need to download the OS X El Capitan gfortran 6.1 binaries regardless of whether or not you are on macOS Sierra, which presently only offers gfortran 6.3.
  4. Downloaded and extracted clang as follows in terminal: 
    curl -OL http://r.research.att.com/libs/clang-4.0.0-darwin15.6-Release.tar.gz 
tar fvxz clang-4.0.0-darwin15.6-Release.tar.gz -C /
  5. Reset the Makevars file in terminal: 
    6. cat <<- EOF > ~/.R/Makevars
# The following statements are required to use the clang4 binary
CC=/usr/local/clang4/bin/clang 
CXX=/usr/local/clang4/bin/clang++
LDFLAGS=-L/usr/local/clang4/lib
# End clang4 inclusion statements
EOF
